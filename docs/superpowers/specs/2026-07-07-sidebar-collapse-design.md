# 文档目录树折叠与自动展开设计

## 背景

MindOS 前端「文档」页左侧目录树当前默认全部展开。当知识库目录层级较深、文件较多时，首屏信息过于密集，用户需要频繁滚动才能定位文件。

## 目标

- 文档页加载时，目录树默认全部折叠。
- 用户仍可手动点击文件夹展开或折叠。
- 从「搜索」或「问答」页点击某条结果/来源跳转到文档页时，自动展开目标文件所在的路径，其他目录保持折叠。

## 范围

仅修改 `src/templates/index.html` 前端逻辑，后端接口与数据格式保持不变。

## 设计

### 1. 默认状态

`renderTree` 创建目录节点时，不再默认添加 `open` 类。子目录容器初始 `display: none`，与 `tree-dir.open` 不添加时 CSS 行为一致。

### 2. 目录节点标识

为每个目录节点增加 `data-path` 属性，值为该目录的相对路径（例如 `docs/api`），用于后续按路径精确展开。

### 3. 路径展开函数

新增 `expandToPath(filePath)`：

1. 将文件路径按 `/` 切分，逐级拼接出父目录路径。
2. 在目录树中查找 `data-path` 匹配的目录节点。
3. 对每个匹配的目录节点添加 `open` 类，并将其子容器 `display` 置为空。
4. 可选：滚动左侧树面板，使选中的文件节点可见。

### 4. 搜索页跳转

- 每条搜索结果渲染为可点击卡片。
- 点击时调用 `showTab('docs')`，随后 `loadTree()` 重新渲染目录树。
- 树渲染完成后调用 `expandToPath(result.path)`，再调用 `loadFile(result.path)` 并高亮对应文件节点。

### 5. 问答页跳转

- 答案来源条目（`.source-item`）点击行为与搜索页一致。
- 当前代码已能跳转并加载文件，只需补充 `expandToPath(s.path)`。

## 交互示例

知识库结构：

```
docs/
  guide/
    quickstart.md
  api/
    auth.md
```

- 首次进入文档页：`docs`、`guide`、`api` 均折叠。
- 用户在搜索页点击 `auth.md` 的结果：左侧自动展开 `docs` → `api`，加载 `auth.md` 内容并高亮该文件。

## 边界情况

- 目标文件位于根目录：无需展开任何目录，直接加载文件。
- 文件路径在树中不存在（如文件已删除）：静默失败，仅切换回文档页。
- 用户手动展开的目录在重新加载树后会重置为折叠（当前行为已如此，因为树在 tab 切换时重新渲染）。

## 测试要点

- 文档页首次加载所有目录折叠。
- 手动点击目录可展开/折叠。
- 搜索页点击结果后目标路径展开并加载文件。
- 问答页点击来源后目标路径展开并加载文件。
- 根目录文件跳转时不报错。
